# Activity Filter Configuration

The Discord bot supports filtering which types of Jules session activities are sent to Discord channels. This helps reduce channel clutter while keeping users informed about important session events.

## Activity Types

### Session Status
- **`sessionCompleted`** - Sent when a Jules session finishes successfully
- **`sessionFailed`** - Sent when a Jules session ends with an error

### Plan Management
- **`planGenerated`** - Sent when Jules creates a plan for the session
- **`planApproved`** - Sent when a session plan is approved

### Progress Updates
- **`progressUpdated`** - Sent for general progress update messages

### Communication
- **`agentMessaged`** - Sent for messages from the Jules agent
- **`userMessaged`** - Sent for user messages to Jules

### Code Changes
- **`changeSet`** - Sent when Jules makes code changes (includes git patches)

### Artifacts
- **`media`** - Sent for media files generated by Jules
- **`bashOutput`** - Sent for command outputs and terminal results

## Configuration

Edit the `activityFilter` array in your bot configuration:

```typescript
// Default: Only session status updates
activityFilter: ['sessionCompleted', 'sessionFailed']

// No filter: All activities
activityFilter: [
  'sessionCompleted',
  'sessionFailed',
  'planGenerated',
  'planApproved',
  'progressUpdated',
  'agentMessaged',
  'userMessaged',
  'changeSet',
  'media',
  'bashOutput'
]

// Minimal filter: Only critical updates
activityFilter: ['sessionCompleted', 'sessionFailed', 'planGenerated']

// Verbose filter: Everything except command outputs
activityFilter: [
  'sessionCompleted',
  'sessionFailed',
  'planGenerated',
  'planApproved',
  'progressUpdated',
  'agentMessaged',
  'userMessaged',
  'changeSet',
  'media'
]
```

## Behavior

- **Activities are always logged** in server logs regardless of filter settings
- **Only Discord notifications are filtered** - all activities continue to be stored in the database
- **Session completion/failure notifications are always sent** by default
- **Filter changes require bot restart** to take effect

## Examples

```typescript
// Quiet mode (default)
activityFilter: ['sessionCompleted', 'sessionFailed']

// Development mode - see everything
activityFilter: [
  'sessionCompleted',
  'sessionFailed',
  'planGenerated',
  'planApproved',
  'progressUpdated',
  'agentMessaged',
  'userMessaged',
  'changeSet',
  'media',
  'bashOutput'
]

// Code-focused mode
activityFilter: ['sessionCompleted', 'sessionFailed', 'changeSet', 'bashOutput']
```
